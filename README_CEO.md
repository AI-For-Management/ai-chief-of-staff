# AI Chief of Staff — 使用说明书

> 这是一份面向非技术人员的完整使用指南。您不需要懂代码，只需按步骤操作即可。

---

## 一、这个系统能帮你做什么？

简单来说，这是一个**AI办公室主任**，它7×24小时运行在你的电脑上，帮你做五件事：

| 功能 | 它做什么 | 你需要做什么 |
|------|----------|-------------|
| 📰 **每日情报** | 每天早上8点自动搜索行业新闻，整理成一份简报发到飞书群 | 告诉它你关注哪些话题 |
| 📋 **任务派发** | 你说一句"下月要上线新产品"，它自动拆成10个具体任务、指定负责人和截止日 | 看完方案点"确认"就行 |
| 📚 **知识管理** | 自动监控你的飞书文档变化，建立可搜索的知识库 | 绑定你想监控的文档 |
| 🚨 **风险预警** | 每天两次扫描项目表，发现延期、人员超负荷、群里的负面消息时主动提醒你 | 绑定项目表和工作群 |
| 🔍 **智能问答** | 基于你公司的文档和数据回答问题 | 直接问 |

**核心原则：所有数据都在你自己的电脑上，不会上传到任何地方。**

## 二、使用前的准备工作

### 2.1 你需要准备的东西

| 准备项 | 说明 | 预计耗时 |
|--------|------|----------|
| 飞书管理员权限 | 需要在飞书开放平台创建应用 | 15分钟 |
| DeepSeek API密钥 | AI大脑的"钥匙"，已有SiliconFlow账号 | 已完成 |
| 电脑保持开机 | 系统需要持续运行才能定时执行任务 | — |

### 2.2 创建飞书自建应用（一次性操作，约15分钟）

这一步是让AI秘书获得"进入你飞书办公室的门禁卡"。

**第一步：进入飞书开放平台**
1. 打开浏览器，访问 https://open.feishu.cn
2. 用你的飞书账号登录
3. 点击右上角「创建应用」→ 选择「企业自建应用」

**第二步：填写应用信息**
- 应用名称：`AI办公室主任`（或你喜欢的名字）
- 应用描述：`CEO智能助手，负责情报分析、任务派发和风险预警`
- 图标：随便选一个

**第三步：获取密钥（最重要）**
1. 创建完成后，进入应用详情页
2. 左侧菜单点击「凭证与基础信息」
3. 你会看到两个关键信息：
   - **App ID**：类似 `cli_a5xxxxxxxxxxxxx`
   - **App Secret**：类似 `xxxxxxxxxxxxxxxxxxxxx`（点击显示）
4. **把这两个值复制下来，待会要用**

**第四步：开通应用权限**
1. 左侧菜单点击「权限管理」
2. 搜索并开通以下权限（勾选后点申请）：

| 权限名 | 用途 |
|--------|------|
| `im:message` | 发送消息（情报简报、风险报告） |
| `im:message.group_at_msg` | 读取群消息（风险词分析） |
| `im:chat` | 获取群信息 |
| `bitable:app` | 读写多维表格（任务管理） |
| `docx:document` | 读取云文档（知识库） |
| `wiki:wiki` | 读取知识库Wiki |
| `task:task` | 创建和分配任务 |
| `contact:user.base:readonly` | 读取员工信息（任务指派） |

**第五步：发布应用**
1. 左侧菜单点击「版本管理与发布」
2. 点击「创建版本」→ 填写版本号（如1.0）→ 保存
3. 提交审核（企业内部应用通常自动通过）
4. 审核通过后，在飞书管理后台「应用管理」中启用该应用

**第六步：配置事件订阅（接收飞书消息）**
1. 左侧菜单点击「事件订阅」
2. 请求地址填写：`http://你的电脑IP:8000/webhook/feishu`
   - 如果只在本机用，填 `http://localhost:8000/webhook/feishu`
   - 如果要外网访问，需要内网穿透工具（后面说明）
3. 记下页面上显示的 **Encrypt Key** 和 **Verification Token**

> 💡 **小贴士**：如果你不确定某个步骤，可以先跳过事件订阅（第六步），系统的大部分功能不依赖它。

## 三、系统配置（5分钟）

### 3.1 打开管理后台

在浏览器中访问：**http://localhost:8501**

你会看到一个管理界面，左侧有四个页面：
- 📊 Dashboard — 系统状态和情报简报
- 🔗 飞书配置 — 配置飞书连接
- 📚 知识管理 — 文档搜索和管理
- ✅ 审批中心 — 任务审批

### 3.2 配置飞书连接

1. 点击左侧「🔗 飞书配置」
2. 点击「➕ 添加飞书应用」展开表单
3. 填写：
   - **配置名称**：随便起个名，如「正式环境」
   - **App ID**：粘贴你在飞书开放平台获取的App ID
   - **App Secret**：粘贴App Secret
   - **Encrypt Key**：粘贴事件订阅页面的Encrypt Key（如果有的话）
   - **Verification Token**：粘贴Verification Token（如果有的话）
4. 先点「🔍 测试连通」— 如果显示"连接成功"，说明配置正确
5. 再点「💾 保存」

### 3.3 绑定飞书资产

"资产"就是你要让AI秘书监控和操作的飞书资源。

在飞书配置页面下方，点击「➕ 添加资产绑定」：

**绑定多维表格（用于任务管理和风险扫描）：**
- 资产类型：选 `bitable`
- 资产Token：打开你的多维表格，浏览器地址栏中 `base/` 后面的那串字符就是
  - 例如地址是 `https://xxx.feishu.cn/base/bascXXXXXXXX`，Token就是 `bascXXXXXXXX`
- 表ID：打开具体的表，地址栏中 `table/` 后面的字符
  - 例如 `tblXXXXXXXX`
- 资产名称：如「产品项目管理表」
- 同步Cron：保持默认 `0 */6 * * *`（每6小时同步一次）

**绑定云文档/Wiki（用于知识库）：**
- 资产类型：选 `doc` 或 `wiki`
- 资产Token：打开文档，地址栏中 `docx/` 后面的字符
- 资产名称：如「产品需求文档」

**绑定群聊（用于风险词分析）：**
- 资产类型：选 `chat`
- 资产Token：群聊的chat_id（需要通过飞书API获取，或联系技术人员协助）
- 资产名称：如「产品研发群」

> 💡 **建议**：先绑定1-2个最重要的资产测试，确认工作正常后再逐步添加。

## 四、日常使用

### 4.1 每日情报简报

**自动模式（推荐）：**
- 系统每天早上8:00自动运行
- 搜索你设定的话题，生成结构化简报
- 通过飞书群消息推送给你

**手动触发：**
1. 打开管理后台 http://localhost:8501
2. 点击「📊 Dashboard」
3. 在搜索话题框中输入你关注的领域，每行一个，例如：
   ```
   AI行业动态
   半导体产业
   新能源汽车
   ```
4. （可选）填入飞书群ID，简报会自动发到群里
5. 点击「🚀 立即生成」
6. 等待约30秒，简报会显示在页面上

### 4.2 任务拆解与派发

这是AI秘书最强大的功能——你只需要说一句话，它帮你安排一切。

**操作步骤：**
1. 打开管理后台 → 点击「✅ 审批中心」
2. 在「CEO指令」框中输入你的战略意图，例如：
   - "下个月我们要上线新产品线，请安排落实"
   - "组织下周五的全员季度复盘会议"
   - "启动东南亚市场调研，两周内出报告"
3. （可选）填入多维表格Token和表ID，任务会自动写入你的项目管理表
4. 点击「🚀 拆解任务」
5. AI会自动拆解为具体任务，显示在页面上，包括：
   - 每个任务的名称
   - 建议负责人
   - 建议截止日期
   - 任务之间的依赖关系
   - 优先级
6. 确认方案没问题后，点击「✅ 批准派发」
   - 任务会自动写入你的飞书多维表格
   - 同时在飞书中给对应员工创建任务
7. 如果方案不满意，点击「❌ 拒绝」，可以修改指令重新拆解

**如果配置了飞书群ID：**
- AI会把任务方案以互动卡片的形式发到你的飞书
- 你直接在飞书里点「确认派发」或「拒绝」就行，不用打开管理后台

### 4.3 知识库搜索

当你的飞书文档被系统扫描入库后，你可以用自然语言搜索：

1. 打开管理后台 → 点击「📚 知识管理」
2. 在搜索框输入问题，例如：
   - "上次产品评审的结论是什么"
   - "东南亚市场的竞品分析"
   - "Q3的OKR目标"
3. 系统会返回最相关的文档片段

### 4.4 风险预警

**自动模式：**
- 系统每天9:30和17:30自动扫描：
  - 多维表格中的逾期任务
  - 群聊中的风险关键词（如"严重阻碍"、"无法按期交付"）
- 发现风险时，自动生成报告推送到你的飞书

**手动触发：**
- 访问 http://localhost:8000/docs
- 找到 `/api/agents/alerts/scan` 端点
- 点击「Try it out」→ 「Execute」

## 六、系统管理

### 6.1 启动与停止

**启动系统：**
打开命令行（按 `Win+R`，输入 `cmd` 回车），输入：
```
cd D:\AI-Secretary
docker compose up -d
```
看到6个容器都启动后，系统就在后台运行了。

**停止系统：**
```
cd D:\AI-Secretary
docker compose down
```

**重启系统（出问题时）：**
```
cd D:\AI-Secretary
docker compose restart
```

**查看系统状态：**
```
cd D:\AI-Secretary
docker compose ps
```
所有容器都显示 `Up` 或 `healthy` 就是正常的。

### 6.2 查看运行日志（排查问题时）

```
docker compose logs fastapi --tail 50
```
这会显示最近50条后端日志。把 `fastapi` 换成 `celery-worker` 可以看定时任务的日志。

### 6.3 数据备份

系统的所有数据存储在Docker卷中。备份命令：
```
docker compose exec postgres pg_dump -U chief chief_of_staff > backup.sql
```

## 七、常见问题

### Q: 管理后台打不开？
- 确认Docker Desktop正在运行（系统托盘有鲸鱼图标）
- 在命令行运行 `docker compose ps`，检查是否所有容器都在运行
- 如果fastapi容器不健康，运行 `docker compose restart fastapi`

### Q: 情报简报生成失败，显示"未抓取到新闻"？
- 这是因为Docker容器内的网络不能直接访问Google/DuckDuckGo
- 解决方案：在 `docker-compose.yml` 的 fastapi 服务下添加代理环境变量：
  ```yaml
  environment:
    - HTTP_PROXY=http://host.docker.internal:7890
    - HTTPS_PROXY=http://host.docker.internal:7890
  ```
  （7890是你代理工具的端口号）

### Q: 飞书连接测试失败？
- 确认App ID和App Secret没有复制错（注意前后不要有空格）
- 确认飞书应用已经发布并启用
- 确认应用权限已经审批通过

### Q: 任务拆解结果不够好？
- 给出更具体的指令，比如不说"安排一下"，而说"下个月15号前完成新产品线的需求评审、技术方案和UI设计"
- AI会基于你的描述细节来拆解，信息越多结果越好

### Q: 如何修改情报简报的定时时间？
- 目前默认每天早上8点。如需修改，需要编辑 `app/workers/celery_app.py` 中的 `crontab(hour=8, minute=0)`
- 例如改为下午2点：`crontab(hour=14, minute=0)`
- 修改后运行 `docker compose restart celery-beat`

### Q: 系统占用多少电脑资源？
- 内存：约1-2GB
- CPU：平时几乎不占用，只在执行任务时短暂使用
- 硬盘：约2GB（Docker镜像）+ 数据库增长
- 不会明显影响你的日常电脑使用

### Q: 数据安全吗？
- 所有数据存储在你本地电脑的Docker容器中
- 没有任何数据会上传到第三方服务器
- 唯一的外部通信是：调用DeepSeek API（发送文本，获取AI回复）和调用飞书API（读写飞书数据）
- DeepSeek API传输的是临时查询内容，不会存储你的数据

## 八、系统架构（了解即可）

```
你的电脑
└── Docker Desktop
    ├── 🌐 FastAPI (8000端口)     — AI大脑，处理所有请求
    ├── 🗄️ PostgreSQL (5432端口)  — 数据仓库，存储配置、文档、任务
    ├── ⚡ Redis (6379端口)        — 高速缓存，任务队列
    ├── ⚙️ Celery Worker           — 后台工人，执行耗时任务
    ├── ⏰ Celery Beat             — 闹钟，定时触发任务
    └── 🖥️ Streamlit (8501端口)   — 管理界面，你日常操作的地方
```

AI秘书内部有三个"专员"，各司其职：

| 专员 | 负责什么 | 工作方式 |
|------|----------|----------|
| 🕵️ 情报专员 | 搜索新闻 → 整理分析 → 发送简报 | 每天自动 + 可手动触发 |
| 📋 项目专员 | 拆解任务 → 等你审批 → 写入飞书 → 通知负责人 | 你发指令后执行，写操作前必须你确认 |
| 🚨 风险专员 | 扫描逾期 → 分析群聊情绪 → 生成报告 → 推送预警 | 每天两次自动扫描 |

## 九、技术支持

如果遇到无法解决的问题：

1. **查看日志**：`docker compose logs --tail 100` 会显示最近的系统日志

3. **联系技术团队**：将日志内容和问题描述发给技术支持
