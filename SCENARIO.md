# 端到端测试场景：星辰科技

> 一个虚拟的小型 AI SaaS 初创公司，CEO 通过本系统管理日常运营。本手册引导你跑完所有功能，并在每一步标注**预期结果**和**已知问题**。

## 角色背景

**公司**: 星辰科技
**业务**: AI 客服 SaaS（卖给电商、教育、金融等行业）
**当前阶段**: A 轮后，10 人小团队

**两个项目**:
- **智能客服 Bot v2.0**（进行中，3 周内必须上线）
- **客户数据看板**（规划中）

---

## 第 0 步：准备数据

```bash
# 在项目根目录执行
docker compose exec fastapi python scripts/seed_demo.py
```

或者从宿主机：

```bash
cd D:\AI-Secretary
python scripts/seed_demo.py
```

执行后会创建：
- 6 名员工（技术 3 + 产品 2 + 销售 1）
- 2 个项目（含成员、里程碑、最近进展）

**预期看到**：
```
✓ 员工: 张明远 (技术总监)
✓ 员工: 李婧 (高级后端工程师)
... (6 个员工)
✓ 项目: 智能客服Bot v2.0
✓ 项目: 客户数据看板
✓ 进展: 完成技术方案评审...
... (4 条进展)
```

## 第 1 步：登录入口验证

打开 **http://localhost:8501**

**预期看到**：纯登录页，**侧边栏完全消失**，居中显示 "AI 首席参谋"。

输入 `admin` / `admin123` 登录。

**已知潜在问题**：
- ⚠️ 如果设了开机自启又没改密码，密码默认仍是 `admin123`，**不安全**
- ⚠️ 没有验证码、没有登录失败次数限制（可被暴力破解）

---

## 第 2 步：综合面板

登录后落到首页 "控制台"。

**预期看到**：
- 顶部三个状态卡片：数据库正常、缓存服务正常、整体全部就绪
- 左半屏：最新情报（**应该为空**，因为还没生成）
- 右半屏：进行中的项目 → 应该看到「智能客服Bot v2.0」
- 团队概览：6 人，但**贡献分应为 0**（指标还没跑）
- 风险提醒：当前无风险提醒

**已知潜在问题**：
- ⚠️ 综合面板首次加载比较慢（要打 4 个 API），中间可能有空白闪烁
- ⚠️ 风险提醒为空时显示"当前无风险提醒"，但实际可能只是没运行过预警 Agent，会误导

---

## 第 3 步：手动触发员工指标 (HR Agent)

侧边栏点 **人事管理** → **贡献排行榜**

**预期**：暂无排行数据。

打开浏览器访问 **http://localhost:8000/docs**，找 `POST /api/hr/metrics/refresh` 触发后台任务。

**等 30 秒**，回到人事页 → 排行榜应该有数据了（但都是 0 分，因为没有 Bitable 任务源）。

**已知问题暴露**：
- 🐛 **HR Agent 依赖飞书 Bitable**：未绑定时，所有员工指标都是 0
- 🐛 排行榜会列出每个人贡献分 0 的卡片，看起来很滑稽

---

## 第 4 步：员工图谱 (Profile Agent)

人事管理 → 员工列表 → 展开 **张明远**

**预期看到**：基础信息、技能标签，但 "尚未生成图谱"。

点击 **生成/更新图谱** 折叠区，**留空简历框**，直接点 **生成图谱**。

**预期**：
- 转圈 30-60 秒（调用 V4-Pro 强模型）
- 完成后显示：整体画像 + 优势 + 待提升 + 适合的角色 + 成长建议 + 技能详情
- 其中"项目参与"会引用 "智能客服Bot v2.0"（因为有数据）

**已知问题暴露**：
- 🐛 **首次生成耗时长**：30-60 秒没有进度反馈，用户以为卡死
- 🐛 没填简历时 LLM 只能依据数据库现有信息发挥，质量不稳定

---

## 第 5 步：员工对话 Agent

侧边栏点 **员工对话** → 选 **张明远**。

输入：
> 张明远最近工作怎么样？建议给他派什么任务？

**预期**：
- AI 回答会**引用刚生成的图谱**（如果你按顺序操作）
- 提到他参与"智能客服Bot v2.0"
- 给出具体的任务建议

再问一句：
> 他和李婧谁更适合做架构评审？

**预期**：AI 应该比较两人的画像（如果李婧也生成了）。

**已知问题暴露**：
- 🐛 如果只生成了张明远的图谱，提到李婧会"瞎编"（LLM 没数据支撑会幻觉）
- 🐛 多个员工对话切换会**累积所有对话历史**到 LLM context，长对话会触发 token 上限

---

## 第 6 步：智能规划 Agent (重头戏)

侧边栏点 **智能规划**

**输入**:
> 我们的客服Bot需要在三周内上线，请帮我拆解开发计划

**预期**：
- AI 调用 RAG 检索 → 应该能找到刚刚的项目数据
- 输出包含：项目背景、任务分解表、里程碑、风险提示
- 任务分解会**指定具体的人**（张明远、李婧、王浩然）

**继续追问**（多轮对话验证）：
> 把第3个任务的截止日期提前一周

**预期**：AI 修订方案，保留其他任务，只改第3个。

最后点 **批准派发**。

**预期**：
- 由于飞书未配置 → 失败
- 但本地 agent_sessions 应该有记录

**已知问题暴露**：
- 🐛 **规划 Agent 不知道当前日期**，可能给出"6 月 15 日"这种过去的日期
- 🐛 RAG 检索的 top_k=5 在文档少时全部返回，可能塞太多无关上下文
- 🐛 批准派发依赖飞书 Bitable，未配置时**没有友好错误提示**

---

## 第 7 步：情报简报 Agent

侧边栏点 **情报简报**

输入话题：
```
AI客服行业动态
大模型产业新闻
```

点击 **立即生成**。

**预期**：
- 抓取新闻 + LLM 总结
- 应该**结合公司项目**："对智能客服Bot v2.0 的影响..."
- 自动入库（控制台首页"最新情报"会更新）

**已知问题暴露**：
- 🐛 **DuckDuckGo 在中国大陆不可访问**，没代理时降级到搜狗
- 🐛 搜狗搜索结果质量比 DDG 差很多
- 🐛 一次抓 8 条新闻 × 多个话题，**可能抓不到**（被反爬）

---

## 第 8 步：知识库

侧边栏点 **知识库**

搜索：
> 智能客服项目

**预期**：能搜到刚才生成的情报简报和项目相关内容。

搜索：
> 张明远的能力

**预期**：能搜到张明远的能力画像（第 4 步入库的）。

**已知问题暴露**：
- 🐛 文档少时相关度阈值没设置好，搜啥都能"匹配"出无关文档
- 🐛 简报+画像+HR摘要混在一起，**没有分类筛选**（虽然 metadata 有，但 UI 没暴露）

---

## 第 9 步：风险预警 Agent

打开 **http://localhost:8000/docs** → `POST /api/agents/alerts/scan`

**预期**：
- 扫描 Bitable 逾期任务（飞书未配置 → 0 条）
- 扫描群聊风险词（飞书未配置 → 0 条）
- risk_level 为 low → **不入库不推送**

**已知问题暴露**：
- 🐛 **预警 Agent 强依赖飞书**：不绑定 Bitable + 群聊就完全没用
- 🐛 风险词是硬编码（"严重阻碍"、"延期"等），不能让用户自定义

---

## 第 10 步：知识库管理员 Agent

`POST /api/agents/kb/manage`

**预期**：
- 扫描全部已入库文档
- 找出疑似重复（首次跑应该 0 对）
- 给所有文档打分类标签（briefing→情报、employee-profile→员工 等）
- 生成报告本身入库

**已知问题暴露**：
- 🐛 同一篇文档重复入库（如重复运行第 4 步生成图谱）会触发"假阳性重复"
- 🐛 距离阈值 0.20 是经验值，可能漏掉真正的重复

---

## 测试结束 — 系统能力总结

### 完整跑通的能力闭环
1. CEO 给指令 → 智能规划结合知识库拆任务 ✓
2. 情报抓取 → 自动入库 → 综合面板可见 ✓
3. HR 指标 → 综合面板可见、排行榜 ✓
4. 员工图谱 → 入库 → 员工对话引用 ✓
5. 全部内容可在知识库语义搜索到 ✓

### 暴露的核心问题（按优先级）

| 优先级 | 问题 | 后续优化建议 |
|--------|------|-------------|
| P0 | 强依赖飞书：HR/规划/预警 在没绑定时几乎不可用 | 增加"演示模式"用本地任务表替代飞书 |
| P0 | LLM 不知今天日期、不知当前公司具体情况 | 在 system prompt 中注入当前日期 + 公司基础信息 |
| P1 | 简报 RAG 检索质量差（文档少时全召回） | 加相关度过滤阈值（>0.3 的才用） |
| P1 | 长时间任务无进度反馈 | Streamlit 用 `st.status` 显示中间步骤 |
| P1 | 登录密码默认值且无尝试次数限制 | 强制首次登录改密 + 失败 5 次锁 5 分钟 |
| P2 | 容器内访问 DDG 失败 | 集成 Tavily/Bochaai 国内可用搜索 API |
| P2 | 知识库 UI 没分类筛选 | 增加"分类"下拉过滤 |
| P2 | 员工对话累积历史导致 token 超限 | 实现摘要压缩或滑动窗口 |

## 故障排除

| 现象 | 排查 |
|------|------|
| `seed_demo.py` 报 connection refused | 容器没起来：`docker compose ps` |
| 创建员工 422 错误 | 工号已存在：先 `DELETE /api/hr/employees/{id}` |
| 图谱生成转圈不结束 | 看后端日志：`docker compose logs fastapi --tail 50` |
| 情报抓不到新闻 | 容器需要代理才能访问外网，参考 docker-compose.yml |
